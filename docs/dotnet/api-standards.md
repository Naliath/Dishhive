# .NET API Standards — Dishhive

**Created:** April 19, 2026

These standards align with Freezy's API conventions.

---

## Controller Conventions

- Controllers are annotated with `[ApiController]` and `[Route("api/[controller]")]`
- Action methods use attribute routing (`[HttpGet]`, `[HttpPost]`, etc.)
- Return `IActionResult` or `ActionResult<T>` for type-safe responses
- Never return entity models directly — always use DTOs

```csharp
[ApiController]
[Route("api/[controller]")]
public class RecipesController : ControllerBase
{
    [HttpGet]
    public async Task<ActionResult<IEnumerable<RecipeDto>>> GetAll() { ... }

    [HttpGet("{id:guid}")]
    public async Task<ActionResult<RecipeDto>> GetById(Guid id) { ... }

    [HttpPost]
    public async Task<ActionResult<RecipeDto>> Create(CreateRecipeDto dto) { ... }

    [HttpPut("{id:guid}")]
    public async Task<IActionResult> Update(Guid id, UpdateRecipeDto dto) { ... }

    [HttpDelete("{id:guid}")]
    public async Task<IActionResult> Delete(Guid id) { ... }
}
```

## Response Codes

| Scenario | Status Code |
|----------|-------------|
| Successful GET (single or list) | 200 OK |
| Successful POST (creation) | 201 Created + Location header |
| Successful PUT/PATCH | 204 No Content |
| Successful DELETE | 204 No Content |
| Validation failure | 400 Bad Request |
| Resource not found | 404 Not Found |

## DTOs

- DTOs live under `Models/DTOs/`
- Grouped by feature in a single file (e.g., `RecipeDtos.cs`)
- Input DTOs use Data Annotations for validation
- Output DTOs are plain C# records or classes

```csharp
public static class RecipeDtos
{
    public record RecipeDto(Guid Id, string Title, string? Description, int Servings, DateTime CreatedAt);
    public record CreateRecipeDto([Required][MaxLength(200)] string Title, string? Description, int Servings = 4);
    public record UpdateRecipeDto([Required][MaxLength(200)] string Title, string? Description, int Servings);
}
```

## Entity Design

- All entities use `Guid` primary key with `HasDefaultValueSql("gen_random_uuid()")`
- All entities have `CreatedAt` and `UpdatedAt` timestamps
- `UpdatedAt` is automatically set in `SaveChanges` override in `DishhiveDbContext`
- String lengths are always constrained with `HasMaxLength()`

## Service Layer

- Services encapsulate business logic and external calls
- Controller actions should be thin; delegate to services
- Services are registered via `IServiceCollection` extensions

## Environment-Aware Configuration

- PostgreSQL is skipped when `ASPNETCORE_ENVIRONMENT=Testing`
- Test environment uses in-memory EF Core database
- Connection strings are injected via environment variables in Docker

```csharp
if (!builder.Environment.IsEnvironment("Testing"))
{
    builder.Services.AddDbContext<DishhiveDbContext>(options =>
        options.UseNpgsql(builder.Configuration.GetConnectionString("DefaultConnection")));
}
```

## Health Checks

- Endpoint: `GET /health`
- Database health check included in non-test environments
- Used by Docker's `healthcheck` configuration

## Swagger / OpenAPI

- Always enabled (useful for self-hosted apps and development)
- Route prefix: `/swagger`
- Version from `AppVersion.Version` (auto-generated by sync-version script)
